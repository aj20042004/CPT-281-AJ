Answer to Question 9

1)The statement moves the iterator forward for one position.  After executing the statement, current points to the node containing "Dick" and offset goes up to 1.  The statement itself returns the iterator after changes occurred (prefix).

2)The statement inserts a new node containing data "Lisa" at iterator position (exact insertion position is between the nodes containing "Tom" and "Dick").  After executing the statement, current points to the node containing "Dick" and offset goes up to 2.  The num_of_items of the list becomes 5 (one more).

3)The statement moves the iterator forward for two positions.  After executing the statement, current points to the node containing "Pete" and offset goes up to 4.  The statement itself returns the iterator after movements.

4)The statement deletes the element at current iterator position.  Therefore, node containing "Pete" is deleted.  After executing the statement, current points to just after the last element of the list and offset stays with 4 (no change).  The tail of the list points to the node containing "Sam".  num_of_items of the list becomes 4 (one less).

5)The statement let the iterator positioned on the last element of the list.  The current will point to the node containing "Sam" and offset will be set to 3.

6)The statement writes the element at the iterator position to the console.  Specifically, string "Sam" will be written to the console.
